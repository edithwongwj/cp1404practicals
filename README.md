## Clean Code Lessons Learned

Here are three key things I've learned about writing clean code so far:

- **Meaningful Names:** Variable and function names should clearly describe their purpose
- **Single Responsibility Principle (SRP):** A function or a class should only have one job
- **Don't Repeat Yourself (DRY):** If I find myself copying and pasting code, I should probably make it into a reusable
  function
